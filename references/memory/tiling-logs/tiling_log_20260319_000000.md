## Failure Analysis

### Program memory overflow (19 failures)
- **Trigger**: m=16 or n=16 on shapes ≥128, generating very large total pipeline stage counts
- **Root cause**: `[AIE ERROR] _XAie_LoadProgMemSection():231: Overflow of program memory`
- **Shapes affected**: 64×64×64, 64×64×128, 128×128×128, 256×256×256, 256×256×512, 512×256×512, 512×512×256, 1024×768×768
- **Key examples**:
  - (256,256,256) i8 tile=(16,16,64): 16×16×4=1024 stages → failed
  - (512,256,512) i8 tile=(16,16,64): 32×16×8=4096 stages → failed
  - (512,512,256) bf16 tile=(16,16,64): 32×32×4=4096 stages → failed

### UnresolvableMapping (6 failures)
- **Trigger**: n=128 on Np=768 → N//n=6 (not power of 2)
- **Shape**: (1024,768,768) i8
- **Failing tiles**: (32,128,128), (128,128,32), (16,128,128), (32,128,64), (64,128,32), (128,128,16)
- **Safe n values for N=768**: n=32 (N//n=24), n=64 (N//n=12) — both pass despite non-power-of-2 quotients

## Key Insights Discovered

1. **(1024,768,768) i8 — major tile discovery**: Prior data had only tested small symmetric tiles (64,64,64 being best at 445.9µs). Tile (128,64,64) at 393µs is 11.9% faster. Large m=128 is highly effective on 1024-M shapes, and n=64 is safe for N=768 (quotient 12, valid despite non-power-of-2).

2. **Small tiles (m=16, n=16) reliably fail on medium/large shapes**: Total pipeline stage count exceeds AIE program memory capacity. Do not enumerate m=16 or n=16 on shapes ≥256. Prefer m,n ≥ 64 on 1024+ shapes.

3. **n=128 on N=768 always fails (6 confirmed examples)**: N//n=6 triggers UnresolvableMapping. n=32 and n=64 are safe. This is a firm enumeration rule.

4. **(64,64,64) on 64×64×64 remains hard to beat**: 14 novel tiles tested, all slower. Prior best (64,64,32) @ 114.4µs stands.
